As I am using Linux (Ubuntu 25.04), the steps will be different from the usual windowns installation. 

\section{Installation}
\begin{lstlisting}
sudo apt install postgresql
\end{lstlisting}

Now, you might have role issues, like I did and get an error, such as :
\begin{lstlisting}
psql: error: connection to server on socket "/var/run/postgresql/.s.PGSQL.5432" failed: FATAL: role "<your_username>" does not exist
\end{lstlisting}

In which case, run the following commands:
\begin{lstlisting}
sudo -i -u postgres
psql
CREATE ROLE <your_username> WITH LOGIN PASSWORD 'your_password';
ALTER ROLE <your_username> WITH SUPERUSER;
\end{lstlisting}

After that, your user should be able to connect to the psql service with:
\begin{lstlisting}
psql -U <your_username> -d postgres
\end{lstlisting}
